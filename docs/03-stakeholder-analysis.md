# 03. Stakeholder Analysis

| Stakeholder Role | Description | Primary System Needs |
| :--- | :--- | :--- |
| **End User (Consumer)** | Everyday individuals managing personal subscription profiles. | Rapid item entry, real-time calculation changes, and high mobile responsiveness. |
| **Course Assessor** | Academic evaluator reviewing system requirements execution. | Clean web engineering practices, proper data isolation, and clear client-server request logic. |
